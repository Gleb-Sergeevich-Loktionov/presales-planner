## 3. Defects

### CRITICAL

**C1 — `/dev-login` grants full admin, zero auth** (LIVE-PROVEN)
`GET /dev-login` → `303` + admin cookie. Gated only by `debug`, and `.env` has `DEBUG=true`; server binds `0.0.0.0`. Any device on the network → 12h admin.
*Cause:* `routes/auth.py:32`. *Fix:* delete/loopback-gate; never expose with `DEBUG=true`.

**C2 — Forgeable admin JWT** (LIVE-PROVEN)
`JWT_SECRET=dev-insecure-change-me` (public default). Forged `{"is_admin":true}` token offline; decoded the live cookie. Independent of C1.
*Cause:* `settings.py:38`. *Fix:* random secret; fail startup on the known default; rotate.

### HIGH

**H1 — No rate-limiting + no user allowlist** (code-proven)
Bot answers every private message; each text → Claude API call. No throttle/allowlist anywhere. Stranger can spam → unbounded API spend + DB writes.
*Fix:* throttling middleware + sender-is-known-person gate.

**H2 — `capture_task` writes before the admin gate** (code + DB-proven)
`CaptureTaskIntent` handled before `can_execute`; excluded from `WRITE_KINDS`. Auto-creates projects from arbitrary names. Any sender writes projects/tasks.
*Fix:* require known team member (couple with H1); decide policy on open capture.

## 4. Security summary

- ✅ **No SQL injection** — parameterized SQLAlchemy throughout.
- ✅ **No stored XSS** — Jinja2 autoescape on (`<script>` → `&lt;script&gt;`).
- ✅ **Secrets not in git** — `.env` gitignored, clean history.
- ⚠️ **Prompt injection bounded** by the Pydantic `Intent` union; worst case is steering classification, gated as normal.
- ❌ **Two admin-bypass paths** (C1, C2) + **open write path** (H2) are the real authz failures.
- CSRF mitigated by `SameSite=lax`.
